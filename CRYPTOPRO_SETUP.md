# Настройка КриптоПро CSP для CISChecker (УКЭП)

## Что нужно для работы УКЭП

Чтобы CISChecker мог подписывать запросы вашим сертификатом УКЭП и автоматически получать JWT-токен, необходимо:

1. **КриптоПро CSP 5.x** (рекомендуется) или 4.x
2. **USB-токен** (RuToken, eToken, JaCarta) с сертификатом УКЭП
3. **Python-библиотека `pywin32`** для доступа к COM-объектам

---

## Пошаговая инструкция

### 1. Установка КриптоПро CSP

1. Скачайте КриптоПро CSP 5.0 R3 с [cryptopro.ru](https://www.cryptopro.ru/products/csp/downloads)
2. Установите с параметрами **по умолчанию**
3. После установки **перезагрузите компьютер**
4. Активируйте лицензию (можно использовать пробную на 90 дней)

**Проверка установки:**
- Откройте **Пуск → КриптоПро CSP** (панель управления КриптоПро)
- На вкладке **Общие** должна быть версия 5.x.xxxxx

### 2. Установка сертификата УКЭП

1. Подключите USB-токен (RuToken/eToken) к компьютеру
2. Откройте **КриптоПро CSP → Сервис → Просмотреть сертификаты в контейнере**
3. Нажмите **Обзор** → выберите ваш контейнер (обычно RuToken/eToken)
4. Нажмите **Далее** → **Установить**
5. Выберите хранилище **«Личные» (My)** — **ЭТО КРИТИЧЕСКИ ВАЖНО!**
6. Нажмите **ОК**

**Проверка:**
- Откройте **КриптоПро CSP → Сервис → Просмотреть сертификаты**
- В списке должен быть ваш сертификат с ИНН

### 3. Проверка в браузере (опционально)

1. Откройте https://markirovka.crpt.ru
2. Нажмите **Войти по сертификату**
3. Если браузер видит сертификат — КриптоПро настроен правильно

### 4. Установка pywin32 (для Python)

```cmd
pip install pywin32
```

Если используете виртуальное окружение:
```cmd
.venv\Scripts\activate
pip install pywin32
```

### 5. Настройка CISChecker

1. Запустите CISChecker (`python gui_app.py` или `CISChecker.exe`)
2. Перейдите в **Настройки → 🔐 Получить токен через УКЭП**
3. Нажмите **🔄 Обновить список** — должны появиться сертификаты
4. Выберите нужный сертификат
5. Нажмите **🔐 Получить токен**

## Решение проблем

### Сертификаты не появляются в CISChecker

**Шаг 1: Запустите диагностику**
В окне УКЭП нажмите кнопку **🔍 Диагностика COM**. В логе появится список доступных COM-объектов:

```
✅ CAdESCOM.Store — доступен           ← КриптоПро CSP 5.x работает
✅ CAPICOM.Store — доступен            ← Windows Certificate Store
❌ CPCSPStore.Store — НЕДОСТУПЕН       ← это нормально для CSP 5.x (legacy 4.x)
✅ CAdESCOM.CPSigner — доступен       ← подпись доступна
✅ CAdESCOM.CadesSignedData — доступен ← данные для подписи доступны
✅ CAdESCOM.About — доступен
   Версия КриптоПро: 2.0.xxxxx        ← версия CAdESCOM плагина (не CSP!)
```

**Шаг 2: Проверьте результаты**

| Результат | Причина | Решение |
|-----------|---------|---------|
| Все ❌ | КриптоПро CSP не установлен | Установите CSP 5.x и перезагрузите |
| `CAdESCOM.Store ✅` но 0 сертификатов | Сертификат не в хранилище «Личные» | Установите сертификат через КриптоПро CSP (шаг 2) |
| `pywin32 НЕ установлен` | Нет COM-доступа | `pip install pywin32` |
| Ошибка `Class not registered` | COM-объекты не зарегистрированы | Переустановите КриптоПро CSP |

**Шаг 3: Проверьте хранилище вручную**

Откройте PowerShell и выполните:
```powershell
certutil -user -store My
```

Если сертификат виден здесь, но не в CISChecker — проблема в COM-доступе.

### Ошибка «Не удалось подписать данные»

1. Убедитесь что USB-токен подключён
2. Попробуйте подписать тестовый файл через КриптоПро CSP:
   - **КриптоПро CSP → Сервис → Подписать файл**
   - Если КриптоПро тоже не может подписать — проблема в токене/сертификате

### Ошибка авторизации (HTTP 400 или 403)

- **HTTP 400**: Неверный формат подписи. Убедитесь что КриптоПро CSP 5.x (не 4.x)
- **HTTP 403**: Пользователь не зарегистрирован в ЧЗ или неактивен
  - Проверьте вход через https://markirovka.crpt.ru (по сертификату)

### USB-токен не определяется

1. Установите драйверы RuToken: [rutoken.ru/support/download](https://www.rutoken.ru/support/download/)
2. Или eToken: используйте SafeNet Authentication Client
3. После установки драйверов перезагрузите компьютер

## FAQ

**Q: Нужна ли лицензия КриптоПро CSP?**
A: Для тестирования можно использовать пробную лицензию (90 дней). Для постоянной работы нужна лицензия.

**Q: Можно ли использовать КриптоПро CSP 4.x?**
A: Да, но рекомендуется 5.x — он лучше поддерживает современные COM-объекты.

**Q: Сколько живёт JWT-токен?**
A: JWT-токен живёт примерно 10 часов. Время истечения отображается в заголовке CISChecker.

**Q: Можно ли использовать УКЭП на Linux?**
A: Текущая версия поддерживает УКЭП только на Windows.
