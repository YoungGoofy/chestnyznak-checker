# CISChecker — Проверка кодов маркировки «Честный Знак»

Проверка кодов маркировки системы **«Честный Знак»** с выгрузкой результатов в Excel.

Два интерфейса:
- **GUI** (`gui_app.py`) — графическое приложение с кнопками, логами и настройками
- **CLI** (`check_codes.py`) — консольный скрипт для автоматизации

Два режима работы:
- **Публичный API** — быстрая проверка без авторизации: статус кода, GTIN, бренд, производитель. Владелец **недоступен**.
- **True API** — полная информация с авторизацией: **текущий владелец**, статус, GTIN, бренд, производитель, дата и способ ввода в оборот.

Результат сохраняется в `.xlsx`-файл с форматированием (заголовки, чередование строк, автофильтр).

---

## Требования

- Python 3.8+
- Доступ к `mobile.api.crpt.ru` и `markirovka.crpt.ru` (нужен российский IP — сервера ЧЗ геоблокируют не-РФ адреса)
- Tkinter (встроен в Python на всех платформах — Windows, Linux, NixOS, macOS)
- Для авторизации через УКЭП: Windows + КриптоПро CSP + RuToken

## Быстрый старт (GUI)

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Запуск GUI
python gui_app.py
```

### Работа в GUI

1. **Настройте токен** — один из двух способов:
   - **Настройки → Токен** — вставьте токен True API вручную (полученный из ЛК Честного Знака)
   - **Настройки → Получить токен через УКЭП** — автоматическая авторизация через сертификат электронной подписи (нужен КриптоПро CSP + УКЭП)
2. **Выберите товарную группу** — выпадающий список в верхней части окна. По умолчанию: «Лёгкая промышленность». Если выбрана неверная ТГ — коды не будут найдены.
3. **Файл → Загрузить коды** или кнопка «📂 Отправить список кодов» — выберите TXT-файл с кодами
4. Нажмите «📂 Отправить список кодов» — запустится проверка:
   - Коды отправляются пачками по 100
   - Прогресс отображается в логах: `[150/500] Обработка...`
   - Можно нажать «⏹ Стоп» для остановки
5. **📥 Выгрузка XLSX** — сохраните результат в Excel или CSV

Приложение не зависает во время проверки — всё работает в фоновом потоке.

## Быстрый старт (CLI)

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Настройка токена (только для True API)
#    Отредактируйте файл .env:
#    CHESTNYZNAK_TOKEN=твой_токен_сюда

python check_codes.py --true codes.txt -o result.xlsx

# 4. Запуск с указанием товарной группы
python check_codes.py --true --pg lp codes.txt -o result.xlsx
```

## Авторизация

### Способ 1 — Токен из личного кабинета

1. Войди в ЛК Честного Знака: https://markirovka.crpt.ru
2. Открой консоль браузера: `F12` → вкладка **Console**
3. Выполни команду:
   ```js
   copy(localStorage.getItem('token'))
   ```
4. Токен скопирован в буфер обмена. Вставь в GUI (Настройки → Токен) или в `.env`:
   ```
   CHESTNYZNAK_TOKEN=eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...
   ```

### Способ 2 — УКЭП (русская электронная подпись)

Только для Windows с установленным КриптоПро CSP:

1. Подключи USB-токен (RuToken) с сертификатом УКЭП
2. В GUI: **Настройки → Получить токен через УКЭП**
3. Нажми «🔄 Обновить список» — появится список сертификатов
4. Выбери сертификат и нажми «🔐 Получить токен»
5. Токен сохранится автоматически

### ⚠️ Важно

- Токен живёт **8–12 часов**. После истечения нужно получить новый.
- Если скрипт пишет `Токен недействителен (HTTP 401)` — токен просрочен, обнови.
- УКЭП-авторизация работает только на Windows с КриптоПро CSP.

## Использование (CLI)

### Публичный API (без токена)

Быстрая проверка. Покажет: статус кода, GTIN, бренд, производителя. Владельца **нет**.

```bash
# Проверка одного кода
python check_codes.py --public 0102901036818059215!bNNnpMl)pYo

# Из файла
python check_codes.py --public codes.txt -o public_result.xlsx

# Из stdin
cat codes.txt | python check_codes.py --public --stdin -o result.xlsx
```

### True API (с токеном)

Полные данные: **владелец**, GTIN, бренд, производитель, дата и способ ввода в оборот.

```bash
# Авто-режим (рекомендуется) — сам определяет товарную группу
python check_codes.py --true codes.txt -o result.xlsx

# С указанием товарной группы
python check_codes.py --true --pg lp codes.txt -o result.xlsx
```

### Товарные группы

В GUI доступен выпадающий список с понятными названиями:

| Код ТГ | Название |
|--------|----------|
| `lp` | Лёгкая промышленность *(по умолчанию)* |
| `milk` | Молочная продукция |
| `shoes` | Обувь |
| `tobacco` | Табак |
| `water` | Вода и напитки |
| `beer` | Пиво |
| `perfume` | Парфюмерия |
| `tires` | Шины |
| `medicine` | Лекарства |
| `fur` | Меха |
| `photo` | Фототехника |
| `antiseptic` | Антисептики |
| `beverages` | Безалкогольные напитки |

Полный список кодов ТГ см. в `check_codes.py` — словарь `PRODUCT_GROUPS`.

### Флаги CLI

| Флаг | Описание |
|------|----------|
| `--true` | Использовать True API (нужен токен) |
| `--public` | Использовать публичный API |
| `-f codes.txt` | Файл с кодами (по одному на строку) |
| `--stdin` | Читать коды из stdin |
| `-o result.xlsx` | Путь к выходному файлу (по умолчанию: `result.xlsx`) |
| `--pg lp` | Код товарной группы вручную (только с `--true`) |
| `--token "..."` | Токен напрямую (вместо `.env`) |
| `--delay 0.5` | Задержка между запросами в секундах (по умолчанию: 0.3) |
| `-v` | Показать полный JSON-ответ API в консоли |

## Формат выгрузки (Excel)

10 колонок:

| Колонка | Описание |
|---------|----------|
| **Штрихкод** | Код маркировки (как в исходном файле) |
| **GTIN** | Global Trade Item Number (14 цифр) |
| **Бренд** | Название бренда товара |
| **Индекс картинки** | Идентификатор изображения товара (True API) |
| **Статус** | Статус кода: «В обороте», «Эмитирован», «Продан», «Списан» и т.д. |
| **Количество** | Всегда 1 (один код = одна строка) |
| **Владелец** | Название организации + ИНН (только True API) |
| **Производитель** | Название производителя |
| **Дата ввода** | Дата и время в формате `ДД.ММ.ГГГГ ЧЧ:ММ:СС` (МСК) |
| **Способ ввода** | Производство в РФ / Ввезён в РФ / Маркировка остатков / Перемаркировка |

## Обработка ошибок

При проверке больших списков коды отправляются пачками по 100. В логах отображается прогресс: `[150/500] Отправка пачки 2/5...`.

Если API возвращает ошибку, программа показывает понятное описание:

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Токен недействителен (HTTP 401)` | Токен просрочен (8–12 ч) | Получить новый токен |
| `Доступ запрещён (HTTP 403)` | Нет прав на эту товарную группу | Проверить доступ в ЛК ЧЗ |
| `Слишком много запросов (HTTP 429)` | Превышен лимит запросов | Подождать и повторить |
| `Неверная товарная группа (HTTP 400)` | Выбрана не та ТГ | Выбрать правильную товарную группу |
| `Публичный API недоступен` | Геоблокировка (не российский IP) | Использовать российский IP/VPN |
| `Статус: wrong` | Код повреждён или невалидный | Проверить корректность кода |

## Автообновление

Приложение автоматически проверяет обновления при запуске. Новая версия скачивается с GitHub Releases и заменяет текущий `.exe` при перезапуске.

Текущую версию можно посмотреть в меню **Справка → О программе**.
