# Chestny Znak Code Checker

Проверка кодов маркировки системы **«Честный Знак»** с выгрузкой результатов в Excel.

Два интерфейса:
- **GUI** (`gui_app.py`) — графическое приложение с кнопками, логами и настройками
- **CLI** (`check_codes.py`) — консольный скрипт для автоматизации

Два режима работы:
- **Публичный API** — быстрая проверка без авторизации: статус кода, название товара, производитель. Владелец **недоступен**.
- **True API** — полная информация с авторизацией: **текущий владелец**, статус, производитель, дата и способ ввода в оборот.

Результат сохраняется в `.xlsx`-файл с форматированием (заголовки, чередование строк, автофильтр).

---

## Требования

- Python 3.8+
- Доступ к `mobile.api.crpt.ru` и `markirovka.crpt.ru` (нужен российский IP — сервера ЧЗ геоблокируют не-РФ адреса)
- Tkinter (встроен в Python на всех платформах — Windows, Linux, NixOS, macOS)

## Быстрый старт (GUI)

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Настройка токена — через меню «Настройки → Токен» в GUI
#    или вручную в .env: CHESTNYZNAK_TOKEN=твой_токен

# 3. Запуск GUI
python gui_app.py
```

### Работа в GUI

1. **Настройки → Токен** — вставь токен True API (полученный из ЛК Честного Знака)
2. **Файл → Загрузить коды** или кнопка «📂 Отправить список кодов» — выбери TXT-файл с кодами
3. Нажми «Отправить список кодов» — запустится проверка, прогресс виден в логах
4. **📥 Выгрузка XLSX** — сохрани результат в Excel или CSV

Приложение не зависает во время проверки — всё работает в фоновом потоке. Можно нажать «⏹ Стоп» для остановки.

## Настройка токена (True API)

Для получения **владельца** кода нужен токен True API.

### Как получить токен

1. Войди в личный кабинет Честного Знака: https://markirovka.crpt.ru
2. Открой консоль браузера: `F12` → вкладка **Console**
3. Выполни команду:
   ```js
   copy(localStorage.getItem('token'))
   ```
4. Токен скопирован в буфер обмена.

### Куда вставить токен

**Способ 1 — файл `.env` (рекомендуется):**

Открой файл `.env` в папке проекта и вставь токен:

```
CHESTNYZNAK_TOKEN=eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...
```

**Способ 2 — переменная окружения:**

```bash
export CHESTNYZNAK_TOKEN="eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**Способ 3 — параметр командной строки:**

```bash
python check_codes.py --true --token "eyJ..." codes.txt -o result.xlsx
```

### ⚠️ Важно

- Токен живёт **8–12 часов**. После истечения нужно получить новый (повторить шаги выше).
- Если скрипт пишет `Токен недействителен (HTTP 401)` — токен просрочен, обнови.

## Использование

### Публичный API (без токена)

Быстрая проверка. Покажет: статус кода, товар, производителя. Владельца **нет**.

```bash
# Проверка одного кода
python check_codes.py --public 0102901036818059215!bNNnpMl)pYo

# Из файла
python check_codes.py --public codes.txt -o public_result.xlsx

# Из stdin
cat codes.txt | python check_codes.py --public --stdin -o result.xlsx
```

### True API (с токеном)

Полные данные: **владелец**, производитель, дата и способ ввода в оборот.

**Авто-режим** (рекомендуется) — сам определяет товарную группу:

```bash
# Все коды из файла
python check_codes.py --true codes.txt -o result.xlsx

# Один или несколько кодов напрямую
python check_codes.py --true '0102901036818059215!bNNnpMl)pYo' '0102901036818011215vclXBESBcnl*'
```

**Ручной режим** (если точно знаешь товарную группу):

```bash
# lp — лёгкая промышленность
python check_codes.py --true --pg lp codes.txt -o result.xlsx

# milk — молочная продукция
python check_codes.py --true --pg milk codes.txt -o result.xlsx
```

Коды товарных групп: `lp`, `milk`, `tobacco`, `water`, `beer`, `shoes`, `perfume`, `tires`, `medicine` и др.

### Флаги

| Флаг | Описание |
|------|----------|
| `--true` | Использовать True API (нужен токен) |
| `--public` | Использовать публичный API |
| `-f codes.txt` | Файл с кодами (по одному на строку) |
| `--stdin` | Читать коды из stdin |
| `-o result.xlsx` | Путь к выходному Excel-файлу (по умолчанию: `result.xlsx`) |
| `--pg lp` | Код товарной группы вручную (только с `--true`) |
| `--token "..."` | Токен напрямую (вместо `.env`) |
| `--delay 0.5` | Задержка между запросами в секундах (по умолчанию: 0.3) |
| `-v` | Показать полный JSON-ответ API в консоли |

## Формат выгрузки (Excel)

| Колонка | Описание |
|---------|----------|
| **Штрихкод** | Код маркировки (как в исходном файле) |
| **Индекс картинки вида продукции** | Идентификатор изображения товара (если есть в True API) |
| **Статус** | Статус кода: «В обороте», «Эмитирован», «Продан», «Списан», «Выбыл» и т.д. |
| **Количество кодов маркировки** | Всегда 1 (один код = одна строка) |
| **Владелец** | Название организации + ИНН (только True API) |
| **Производитель** | Название производителя |
| **Дата ввода в оборот** | Дата и время в формате `ДД.ММ.ГГГГ ЧЧ:ММ:СС` (МСК) |
| **Способ ввода в оборот** | Производство в РФ / Ввезён в РФ / Маркировка остатков / Перемаркировка |

## Возможные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Токен недействителен (HTTP 401)` | Токен просрочен (8–12 ч) | Получить новый токен в ЛК |
| `Публичный API недоступен` | Геоблокировка (не российский IP) | Использовать российский IP/VPN |
| `ModuleNotFoundError: openpyxl` | Не установлены зависимости | `pip install -r requirements.txt` |
| `Статус: wrong` | Код повреждён или невалидный | Проверить корректность кода |
